# Отчёт по домашней работе: Мониторинг ML в продакшене

## Доработка кода сервиса



| Файл | Что изменено |
|---|---|
| `ml_service/schemas.py` | Добавление валидации к полям |
| `ml_service/model.py` | Защита свойства `features` от обращения к `None`-модели |
| `ml_service/app.py` | Обработка ошибок в `lifespan`, `/predict`, `/updateModel`, логирование |


### Обрабатываемые ситуации

**1. Невалидные типы данных в теле запроса (`POST /predict`)** \
В таком случае Pydantic автоматически возвращает `422 Unprocessable Entity` с описанием ошибки. К примеру, невалидные данные в запросе: `{"age": "abc"}` (строка вместо числа).

**2. Отрицательные значения числовых полей (`POST /predict`)** \
В таком случае, благодаря добавлению валидации к полям, Pydantic выкинет `422` с описанием нарушенного ограничения.

**3. Значение `hours_per_week` > 168 (`POST /predict`)** \
В таком случае, благодаря добавлению валидации к `hours_per_week`, Pydantic выкинет `422`, так как в неделе не может быть более 168 часов.

**4. Пустой или состоящий из пробелов `run_id` (`POST /updateModel`)** \
Тут аналогично, благодаря добавленной валидации, Pydantic выкинет `422`.

**5. Несуществующий `run_id` в MLflow (`POST /updateModel`)** \
Тут мы отправляем `404` пользователю. 
При этом старая модель **остаётся в работе**, так как `MODEL.data` не обновляется, если `load_model` бросает исключение.

**6. MLflow-сервер недоступен при обновлении модели (`POST /updateModel`)** \
Выбрасывается `503` с описанием ошибки подключения, при этом старая модель остаётся в работе.

**7. Ошибка загрузки модели при старте сервиса** \
В `lifespan` в `app.py` добавлена обработка ошибок конфигурации и загрузки модели с логированием всех проблем.

**8. Модель не загружена при обращении к `/predict`** \
В таком случае, сервер выкинет `503` с информацией о недоступности модели.

**9. Модель не загружена при обращении к свойству `features`** \
В таком случае, на уровне model.py выбрасывается `RuntimeError`, которая в сервисе уже отправляет пользователю `503`.

**10. Ошибка при построении DataFrame из запроса** \
В таком случае пользователю отправляется `422` с описанием ошибки.

**11. Ошибка при вызове `predict_proba`** \
В таком случае пользователю отправляется `500` с деталями ошибки.
